1-of-20 Analog Demultiplexer

Transistor-Level Design · SkyWater 130nm · xschem · Mixed-Signal ASIC
Back to Portfolio

Project Overview

As part of a larger mixed-signal ASIC team project, I designed and simulated a 1-of-20 analog demultiplexer from the transistor level up. The design was implemented in the SkyWater 130nm PDK using xschem for schematic capture, with a 1.8V supply rail. I owned the full implementation of this block — from the individual CMOS transmission gate switch, through the 5-input NAND-based decoder, up to the top-level integration of all 20 channels.

The demux routes a single analog input to one of 20 output nodes based on a 5-bit digital select signal. Both DC and transient simulations were run to verify correctness and characterize non-idealities including on-resistance and off-state leakage.

Layer 1 — MOSFET Operating Regions

The switch at the heart of this design is a CMOS transmission gate — two MOSFETs in parallel. Understanding when each transistor conducts requires knowing the three operating regions. All analysis uses VDD = 1.8V and Vth ≈ 0.6V.

NMOS Regions

Cutoff: Vgs < Vth — transistor OFF, no current.

Triode: Vgs > Vth AND Vds < Vgs − Vth — acts like a resistor. This is the target region for a pass switch.

Saturation: Vds > Vgs − Vth — current source behavior. Not useful for switching.

PMOS Regions

Cutoff: Vsg < |Vtp| — transistor OFF.

Triode: Vsg > |Vtp| AND VSD small — conducts well. PMOS turns ON when gate is pulled LOW.

Key difference: PMOS passes high voltages well but struggles near 0V. NMOS is the opposite — hence both are needed in parallel.

Layer 2 — CMOS Transmission Gate

An NMOS-only switch cannot pass a high input voltage — as Vin approaches VDD, Vgs drops below Vth and the transistor shuts itself off. A PMOS-only switch has the symmetric problem near 0V. The solution is to place them in parallel with complementary gate drives.

xschem schematic: CMOS transmission gate with PMOS (pfet_01v8) and NMOS (nfet_01v8) in parallel. VPG and VNG carry complementary select signals.

When the select line is HIGH: the NMOS gate is driven HIGH (on) and the PMOS gate is driven LOW (also on) — both conduct simultaneously. This keeps Ron low and flat across the full 0–1.8V signal range. The total on-resistance is the parallel combination: Ron = RDS(pmos) ‖ RDS(nmos). VPB and VNB are body bias pins tied to VDD and GND respectively to keep body diodes reverse-biased.

Layer 3 — 5-Input AND Gate via NAND Tree

Each of the 20 output channels needs a dedicated logic gate that fires for exactly one 5-bit address. In CMOS, a 5-input AND gate is built as a tree of 2-input NANDs followed by a final inversion — NAND gates are cheaper (fewer transistors) than AND gates in standard cell libraries.

5-input AND constructed from 2-input NAND pairs. Each NAND+NAND pair produces a 2-input AND. All partial results feed a final NAND which, after inversion, gives A·B·C·D·E.

The NAND output is active-low: YNAND = A·B·C·D·E = 0 only when all five inputs are 1. After inversion (the P0–P4 inverters in the decoder), the final output is HIGH only when all five select bits match — exactly one address.

Truth table: output is logic 1 unless all five inputs are 1. Inversion at the output converts this to an active-high AND gate.

Layer 4 — 5-to-20 Decoder

A 5-bit decoder can address 25 = 32 outputs, but only the first 20 are needed. Twenty copies of the 5-input AND gate are instantiated. Each copy is wired differently: for input bit positions where the target address has a 0, an inverter is inserted before the AND input; where the address has a 1, the line connects directly. This ensures each AND fires for exactly one unique 5-bit code.

xschem schematic: five 5-NAND cells (n0–n4) with per-bit input inverters (p0–p4, B0–B4). Each row of inverters programs one output address.

All 20 decoder channels instantiated. Each column is one 5-NAND block; the input inverter pattern differs per column to encode each address 0–19.

Top Level — decoder_x5 + switch_x20

The top-level schematic connects the decoder outputs directly to the gate controls of the 20 transmission gates. The decoder asserts exactly one of its 20 outputs HIGH at a time; that signal drives the NMOS gate of the selected switch ON while its complement drives the PMOS gate. All other 19 switches stay open.

Top-level integration: decoder_x5 (left, SEL[4:0] inputs) drives switch_x20 (right) via 20 select lines (n0–n19) and their complements (p0–p19).

switch_x20: twenty CMOS transmission gate instances sharing a common analog input node. Each has independent gate control from the decoder.

Analog switch cell interface: VNB/VPB (body bias), VNG/VPG (gate controls), VS (signal input/output).

Switch Non-Idealities — Ron and Leakage

Real CMOS switches deviate from ideal behavior in two ways that were analyzed through DC and transient simulation.

Reference analysis: ON-state Ron creates a voltage divider error with RLOAD; OFF-state leakage ILKG develops an error voltage Vout = ILKG × RLOAD.

Ron (Switch ON)

The parallel PMOS/NMOS on-resistance forms a voltage divider with RLOAD: Vout = Vin · RLOAD / (Ron + RLOAD). Ron variation with signal voltage also causes distortion, which cannot be corrected by calibration.

Leakage (Switch OFF)

Off-state leakage current ILKG flows through RLOAD and creates a spurious output voltage even on unselected channels: Vout,error = ILKG × RLOAD. High-impedance loads worsen this effect.

Common Interview Questions

Q: Why use a CMOS transmission gate instead of just an NMOS switch?
An NMOS alone cannot pass a high input voltage — as Vin approaches VDD, Vgs drops below Vth and the transistor turns off. PMOS has the symmetric problem near ground. The transmission gate puts both in parallel with complementary gate signals so at any signal voltage, at least one device is strongly on. This keeps Ron low and signal-independent across the full 0–1.8V range.
Q: How does the decoder ensure only one output is asserted at a time?
Each of the 20 AND gates is wired to respond to exactly one unique 5-bit code by selectively inverting the input lines. For a given SEL[4:0], only the AND gate whose inversion pattern matches that code will see all-ones at its inputs and output HIGH. All others see at least one zero and stay LOW. The binary encoding of 20 addresses across 5 bits guarantees uniqueness for inputs 0–19.
Q: What limits the analog bandwidth of this demux?
Two main factors: the Ron of the selected transmission gate in series with the load forms an RC low-pass filter with the parasitic capacitances at the output node. Additionally, all 19 unselected switches present off-state drain capacitances that load the shared input node. Wider transistors reduce Ron but increase capacitance — a direct tradeoff for the bandwidth vs. on-resistance specification.
Q: Why are VPB and VNB separate pins on the switch cell?
VPB (PMOS body/well bias) is tied to VDD and VNB (NMOS body bias) to GND to keep the source-body and drain-body junctions reverse-biased. This prevents the body diodes from conducting and eliminates the body effect on Vth, which would otherwise shift the threshold voltage as the signal voltage changes and degrade linearity.
Q: What is Ron modulation and why is it harder to fix than gain error?
Ron modulation means Ron varies with the instantaneous signal voltage Vin. Because Ron forms a voltage divider with RLOAD, a signal-dependent Ron introduces a non-linear gain — distortion. Flat gain error (constant Ron offset) can be removed with a system-level gain trim. Distortion from Ron variation cannot be calibrated out because it changes with the signal itself.
